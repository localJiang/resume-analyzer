# /design — 设计工作流

## 用途
创建 UI/UX 设计、组件架构和视觉规范。

## 使用场景
- 实现新的 UI 组件之前
- 设计新页面或布局时
- 改善用户体验时
- 创建设计系统组件时

## 工作流

### 第 1 步：收集需求
1. 阅读 `memory/ui.md` 了解设计系统指南
2. 阅读 `memory/project.md` 了解项目上下文
3. 审查来自 PM Agent 的用户故事
4. 检查代码库中现有的设计模式

### 第 2 步：用户流程设计
1. 绘制用户旅程（入口 → 目标 → 出口）
2. 识别所有状态（加载、空、错误、成功、边缘情况）
3. 定义交互（点击、悬停、键盘、触摸）
4. 考虑响应式行为（移动端、平板、桌面端）

### 第 3 步：组件设计
1. 将 UI 拆解为组件层级
2. 识别可复用组件与一次性组件
3. 为每个组件定义 Props 接口
4. 指定加载/空/错误状态
5. 设计无障碍（WCAG 2.1 AA）

### 第 4 步：视觉设计
1. 使用 Tailwind + shadcn/ui 设计令牌
2. 遵循 Tailwind 工具优先方法
3. 确保颜色对比度达标
4. 设计响应式断点
5. 添加微交互和过渡效果

### 第 5 步：文档化
1. 创建组件规范
2. 如果引入了新模式，添加到 `memory/ui.md`
3. 引用设计决策

## 输出格式

```markdown
# 设计：[功能/组件名称]

## 用户流程
[步骤及状态描述]

## 组件树
```
Page
├── Header
│   └── ActionButton
├── Content
│   ├── DataCard（×N）
│   └── EmptyState（条件渲染）
└── Footer
```

## 组件规范
### DataCard
- **Props：** data, isLoading, onAction
- **状态：** 默认、加载中（骨架屏）、空、错误
- **响应式：** 移动端 1 列、平板 2 列、桌面端 3 列
- **无障碍：** 键盘可导航、aria-labels

## 视觉规范
- **颜色：** primary-500 用于操作、gray-200 用于边框
- **间距：** 卡片间 gap-4
- **排版：** text-lg 用于标题、text-sm 用于元数据
```

## 约束条件
- 必须以 Tailwind + shadcn/ui 组件为基础
- 必须使用 Tailwind 工具类（禁止内联样式）
- 所有交互元素必须支持键盘访问
- 移动端优先设计
- 不要重新发明 Tailwind + shadcn/ui 中已有的组件
