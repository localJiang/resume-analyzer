# 技能：部署

## 描述
将应用部署到 Vercel 并管理发布流水线。

## 适用场景
- 部署到生产/预发布/预览环境
- 设置 CI/CD 流水线
- 配置环境变量
- 管理生产环境的数据库迁移
- 回滚有问题的部署
- 监控部署后健康状况
- 配置自定义域名和 SSL

## 输入
- 已审批、准备部署的 PR
- 环境配置
- 数据库迁移状态
- 发布说明

## 输出
- 已部署的应用
- 部署日志
- 健康检查确认
- 发布通知

## 约束
- 周五不部署（关键热修复除外）
- 部署前始终准备好回滚方案
- 数据库迁移必须先在预发布环境测试
- 生产部署后监控 30 分钟
- 所有环境变量必须在部署前设置完毕
- 部署必须通过 GitHub Actions 自动化执行（禁止手动 FTP/SSH）

## 部署流水线

```
1. PR 合并 → 触发 CI
2. Lint → 类型检查 → 测试 → 构建
3. 部署预览环境 (per-PR) / 预发布环境 (develop) / 生产环境 (main)
4. 运行数据库迁移
5. 健康检查
6. 冒烟测试
7. 通知团队
```

## 最佳实践
1. **全面自动化** — 无手动部署步骤
2. **不可变部署** — 每次部署创建新实例，不修改现有实例
3. **功能开关** — 将部署与发布解耦
4. **蓝绿部署或金丝雀发布** — 降低不良部署的风险
5. **监控回滚时间** — 应小于 5 分钟
6. **保持预发布 = 生产** — 相同配置，不同规模
