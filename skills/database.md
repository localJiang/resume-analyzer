# 技能：数据库

## 描述
使用 PostgreSQL 配合 Prisma 设计和管理数据库模式、迁移及查询。

## 适用场景
- 设计新表或关系
- 编写数据库迁移
- 优化慢查询
- 创建种子数据
- 数据建模决策
- 性能问题排查

## 输入
- PM/Architect 提供的数据需求
- Backend Agent 提供的查询模式
- 性能要求
- 数据保留策略

## 输出
- 模式定义（Prisma 模式）
- 迁移文件
- 种子脚本
- 索引建议
- 查询优化建议
- ERD 文档

## 约束
- 始终使用迁移，禁止手动编辑数据库
- 每张表必须有 `id`、`created_at`、`updated_at` 字段
- 所有关系使用外键
- 为 WHERE、JOIN、ORDER BY 中使用的列创建索引
- 前端禁止直接访问数据库
- 多表写入使用事务
- 敏感数据静态加密
- 迁移尽量保持可逆

## 最佳实践
1. **为查询设计** — 模式应匹配数据访问模式
2. **先规范化** — 仅在性能需要时才反规范化
3. **策略性索引** — 过多索引会损害写入性能
4. **使用 UUID 作为主键** — 避免序列争用，更适合分布式系统
5. **软删除** — 重要数据使用 `deleted_at` 而非硬删除
6. **测试迁移** — 上线前务必在预发布环境测试
