# 技能：AI 集成

## 描述
将 AI/LLM 能力集成到应用中（聊天、生成、分析、嵌入向量）。

## 适用场景
- 添加 AI 聊天或 Copilot 功能
- 实现文本生成或摘要
- 使用嵌入向量构建语义搜索
- 实现 AI 辅助内容审核
- 构建推荐系统
- 创建智能自动化工作流

## 输入
- PM Agent 提供的功能需求
- AI 端点的 API 契约
- 模型选择标准（成本、延迟、质量）
- 数据隐私和安全要求

## 输出
- AI 服务集成代码
- 提示词模板及其管理
- 流式响应处理
- AI 调用的限流与错误处理
- 成本追踪与优化
- AI 功能测试（使用模拟的 AI 响应）

## 约束
- 绝不在客户端暴露 API 密钥
- 展示前始终校验和清洗 AI 输出
- 为 AI 端点实现限流（成本控制）
- 优雅处理 AI 超时（降级响应）
- 尊重用户数据隐私 — 不要不必要地向 AI 模型发送 PII
- 记录 AI 调用日志以便调试和成本追踪
- 在 AI 处理期间提供用户反馈（流式传输、加载指示器）

## 架构模式

```
客户端 → 你的 API → AI 服务（仅限服务端）→ AI 模型
                         ↑
                    提示词模板
                    上下文组装
                    响应校验
                    成本追踪
```

## 最佳实践
1. **仅服务端调用 AI** — 绝不在客户端直接调用 AI API
2. **提示词模板** — 存储和版本化管理提示词，不要内联书写
3. **校验 AI 输出** — AI 可能产生幻觉；对结构化输出进行校验
4. **流式响应** — 长时间生成时提供更好的用户体验
5. **实现降级方案** — AI 宕机或响应缓慢时如何处理？
6. **成本意识** — 追踪 Token 使用量，设置每用户限额
7. **护栏机制** — 对输入和输出均进行内容过滤
