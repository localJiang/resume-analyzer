# 技能：UI 开发

## 描述
使用 Tailwind + shadcn/ui 和 Tailwind CSS 构建无障碍、响应式的用户界面。

## 适用场景
- 创建新 UI 组件
- 构建页面布局
- 实现设计系统
- 提升无障碍性
- 响应式设计实现
- 动画与微交互
- 深色模式实现

## 输入
- 设计规范或线框图
- 组件需求（状态、属性）
- 设计令牌（颜色、间距、字体）
- 无障碍要求（WCAG 2.1 AA）

## 输出
- 可复用的 UI 组件
- 页面布局
- 响应式样式
- 无障碍标记
- 组件文档

## 约束
- 使用 Tailwind + shadcn/ui 作为基础组件库
- 仅使用 Tailwind 工具类（禁止内联样式，除非必要不使用 CSS 模块）
- 移动优先的响应式设计
- 最低满足 WCAG 2.1 AA 无障碍标准
- 组件必须处理：默认、加载中、空数据、错误、禁用状态
- 文字颜色对比度 ≥ 4.5:1
- 所有可交互元素必须有可见的焦点指示器
- 语义化 HTML 元素（`<button>`，而非 `<div onClick>`）

## 组件状态检查清单

```
✅ 默认    — 正常渲染
✅ 加载中  — 骨架屏或加载指示器
✅ 空数据  — 友好提示及操作引导
✅ 错误    — 错误信息及重试操作
✅ 禁用    — 灰色显示，不可交互
✅ 成功    — 短暂的成功反馈（瞬时显示）
```

## 最佳实践
1. **从移动端开始，向上扩展** — 基础样式为移动端，`md:`、`lg:` 用于更大屏幕
2. **语义化 HTML** — 使用合适元素以确保无障碍性
3. **焦点管理** — 模态框中捕获焦点，关闭时恢复
4. **减少动画** — 尊重 `prefers-reduced-motion` 偏好
5. **触摸目标 ≥ 44px** — 保证移动端可用性
6. **全键盘操作** — 测试 Tab、Enter、Escape、方向键
7. **使用设计令牌** — 颜色从 Tailwind 配置中获取，不硬编码十六进制值
