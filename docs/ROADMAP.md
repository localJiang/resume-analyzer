## 里程碑

### 🎯 MVP（最小可行产品）
**目标时间：** 待定
**目标：** 用户可上传简历并获取 AI 分析报告

| 功能 | 状态 | 优先级 |
|---------|--------|----------|
| 用户认证（注册、登录、退出、密码重置） | 待办 | P0 |
| 简历上传解析（PDF/DOCX/TXT，拖拽+点击） | 待办 | P0 |
| AI 简历分析（四维度 + 评分 + 逐条建议） | 待办 | P0 |
| 分析报告页面（骨架屏加载态 + 结果展示） | 待办 | P0 |
| 工作台（上传区 + 分析历史列表） | 待办 | P0 |
| 首页落地页（Hero + 功能卡片 + CTA） | 待办 | P0 |
| 响应式 UI（移动端 + 桌面端） | 待办 | P0 |
| 空状态、错误状态、加载状态全覆盖 | 待办 | P0 |
| 数据库模式与迁移 | 待办 | P0 |
| 安全审查 | 待办 | P0 |

**MVP 成功标准：**
- [ ] 用户可以创建账户并登录
- [ ] 用户可以上传简历文件并完成 AI 分析
- [ ] 分析报告展示四维度评分和逐条建议
- [ ] 所有页面有加载/空/错误状态处理
- [ ] 零严重 Bug
- [ ] 页面加载时间 < 3s，分析 API p95 < 3s
- [ ] 在移动端、平板、桌面端均可正常使用

## 风险登记册

| 风险 | 缓解措施 | 应急方案 |
|------|-----------|-------------|
| 范围蔓延 | 严格 P0/P1/P2 优先级排序；PM 把控新功能 | 削减 P2 功能 |
| 技术债务 | 每周重构时间；代码审查强制执行标准 | 专门重构 sprint |
| 性能问题 | 持续监控；设定性能预算 | 性能优化 sprint |
| 团队人手不足 | 完善文档；交叉培训 Agent | 缩减范围 |

## 如何更新

1. PM Agent 每周审查路线图
2. 更新功能状态（待办 → 进行中 → 已完成）
3. 根据完成速度调整时间线
4. 将新功能添加到适当的里程碑中
5. 在决策记录中记录决策
6. 每月更新风险评估
